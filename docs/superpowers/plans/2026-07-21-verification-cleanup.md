# Verification Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the migration branch verifiable without changing production behavior outside the TanStack Router migration.

**Architecture:** Update tests whose assertions target retired GameCover and prototype-route behavior so they assert the current public rendering contract. Configure Prettier to tolerate repository CRLF checkouts, then apply formatting only to files changed by this branch.

**Tech Stack:** Vitest, Testing Library, ESLint, Prettier, TypeScript.

## Global Constraints

- Do not add Next.js or restore `next/link` or `next/navigation` imports.
- Do not change runtime application behavior to satisfy stale tests.
- Do not touch `web/.output`.
- Repository-wide formatting is authorized solely to clear the existing ESLint/Prettier baseline failures.

- [ ] Run `rtk proxy npm.cmd --prefix web test -- --run src/features/discovery/discovery.test.tsx`; confirm the old fallback assertion fails before editing and the updated file passes after.
