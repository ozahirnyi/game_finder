# Интеграция дизайна Lovable в основной сайт

## Цель

Перенести весь визуальный интерфейс из `lovable-project-cf1fe460.zip` в
существующий frontend `web/`, сохранив единственное Next.js-приложение,
действующие маршруты, авторизацию и контракты FastAPI.

## Границы

- Источник дизайна: компоненты, дизайн-токены и сценарии из архива Lovable.
- Целевое приложение: Next.js App Router в `web/`.
- Включённые экраны: dashboard, search, library, wishlist, deals, friends,
  Steam, PSN, profile и game detail, а также вход, регистрация и OAuth
  callback в согласованном стиле.
- Не входит: миграция frontend на Vite/TanStack, изменение backend API и
  добавление фиктивных данных в production-интерфейс.

## Архитектура

Lovable-проект не подключается как отдельное приложение. Его presentation
layer переносится в Next.js-компоненты и стили:

- `AppShell`, навигация и общие UI-элементы становятся переиспользуемыми
  компонентами в `web/src/components` и/или `web/src/features`.
- TanStack `createFileRoute`, `Link` и router hooks заменяются на Next
  pages, `next/link` и App Router hooks.
- Маршрут `games.$gameId` сопоставляется с `games/[id]`; library и wishlist
  сопоставляются с текущей моделью saved games и отдельными маршрутами.
- Из архива не переносятся Vite/TanStack entry points, generated route tree,
  Lovable error infrastructure и mock data layer.

## Данные и состояния

- `web/src/lib/api.ts` остаётся единственной точкой доступа к backend.
- Экран использует реальные данные API, где endpoint уже есть: поиск,
  каталог, цены, сохранённые игры, Steam, PSN, профиль и интеграции.
- При отсутствии соответствующего endpoint (в частности для социальных
  Friends/Wishlist-сценариев) показывается явное empty/coming-soon состояние,
  а не данные из макета.
- Каждая область данных имеет loading, error, empty, success и
  unauthenticated состояния. Защищённые запросы не выполняются без токена.
- Изображения из API используются в приоритете. Градиентная обложка служит
  безопасным fallback при отсутствии изображения.

## Визуальная и UX-система

- Переносятся токены цветов, типографика, отступы, карточки, бейджи и
  responsive layout из Lovable-дизайна, адаптированные к текущей кодовой базе.
- Общие действия получают единые компоненты кнопок, форм, панелей и
  уведомлений; интерактивные элементы доступны с клавиатуры и имеют
  видимый focus state.
- Навигация отражает все маршруты, не скрывает существующие возможности и
  корректно меняется на mobile/tablet/desktop.
- Необходимые зависимости добавляются точечно; неиспользуемая большая
  библиотека UI из архива не переносится целиком.

## Проверка

- Компонентные/маршрутные тесты на реальные API-adapter состояния, включая
  ошибки и отсутствие авторизации.
- Frontend: lint, TypeScript и production build.
- Backend: существующий `pytest` suite без регрессий.
- Визуальная проверка основных маршрутов на мобильной, планшетной и desktop
  ширинах, включая keyboard navigation и fallback-изображения.

## Критерии готовности

Пользователь получает единственный Next.js-сайт, где все экраны из дизайна
Lovable доступны по Next-маршрутам, используют реальные возможности проекта
или честные пустые состояния и не нарушают текущую авторизацию либо API.
