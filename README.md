# Game Finder

Навчальний full-stack проєкт: **пошук ігор**, колекції користувача, інтеграція з зовнішнім API, авторизація, кеш і (пізніше) AI-рекомендації. Бекенд пишеться **самостійно на FastAPI**; фронтенд можна збирати з допомогою ШІ (наприклад, Next.js).

Ідея близька до проєкту **movie_finder** (Django + PostgreSQL + зовнішні API), але тут стек — **FastAPI + PostgreSQL + Redis**, домен — **ігри** (наприклад, [RAWG API](https://rawg.io/apidocs)).

---

## Навіщо цей репозиторій

Це наступний крок після **progression_bot** (Telegram-бот, JSON, чиста структура коду): тут з’являються **веб-API**, **БД**, **користувачі**, **кеш** і **зовнішні сервіси** — те, що очікують від junior backend developer.

Проєкт будується **інкрементально**: один репозиторій поступово ускладнюється (не окремі «відірвані» вправи).

---

## Що хочемо зробити (цільовий продукт)

| Область | Що має бути |
|--------|-------------|
| **API** | REST на FastAPI, валідація Pydantic, OpenAPI `/docs` |
| **Дані** | PostgreSQL, SQLAlchemy, міграції Alembic |
| **Користувачі** | Реєстрація / логін, хеш паролів, JWT, захист мутацій |
| **Пошук** | Запити до RAWG (або аналога), нормалізована відповідь для клієнта |
| **Кеш** | Redis, TTL для повторних пошуків |
| **Продукт** | Каталог/колекції (улюблені, «хочу зіграти») — за власним дизайном API |
| **AI (пізніше)** | Ендпоінт рекомендацій через OpenAI (або сумісний API) |
| **Інфра** | Docker Compose (app + postgres + redis), деплой на PaaS/VPS |
| **Фронт** | Next.js (з AI-допомогою): пошук, картка гри, логін, колекції |

---

## Фази розробки (коротко)

1. **Окремий CLI-скрипт** — `httpx` + RAWG, ключ у `.env`, обробка помилок.
2. **FastAPI + in-memory** — CRUD «заглушок» ігор, без БД.
3. **PostgreSQL** — заміна сховища на БД + Alembic.
4. **Авторизація** — користувачі, JWT, власність записів.
5. **RAWG + Redis** — пошук через зовнішній API + кеш.
6. **Бекенд «як у проді»** — AI-ендпоінт, обмеження частоти запитів, Docker, деплой.
7. **Фронт** — UI поверх твого API.
8. **Співбесіди** — алгоритми, SQL, оповідь про архітектуру (трекати прогрес можна в progression_bot).

Детальніше ТЗ і чеклисти — у ClickUp (задачі Game Finder Phase 1–7) або в скрипті `progression_bot/scripts/clickup_create_gamefinder_roadmap.py` (тексти описів фаз).

## Як почати (коли дійдеш до коду в цьому репо)

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt   # з’явиться після ініціалізації FastAPI
cp .env.example .env
# заповни ключі: RAWG, DATABASE_URL, REDIS_URL, SECRET_KEY, …
```

Поки репозиторій порожній окрім README — **Фаза 1** може жити як `scripts/rawg_cli.py`; далі додаси `app/` і compose.
